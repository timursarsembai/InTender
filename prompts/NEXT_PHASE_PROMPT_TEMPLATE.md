# Шаблон следующего этапа

Реализуй этап `<НОМЕР И НАЗВАНИЕ>` из `docs/10-implementation-plan.md`.

До кода прочитай инструкции, исследуй реализацию, создай/обнови execution plan и перечисли acceptance criteria.

Требования:
- законченный вертикальный срез;
- без изменения бизнес-правил;
- транзакционные и идемпотентные денежные действия;
- серверная авторизация;
- миграции, OpenAPI, тесты, документация;
- без критических TODO и преждевременных абстракций;
- запустить lint, typecheck, unit, integration и релевантные e2e.

В финале: изменения, проверки, миграции, ограничения и следующий этап.
