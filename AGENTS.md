# AGENTS.md

## Источник истины

Ты работаешь над InTender. Перед изменением поведения прочитай релевантные документы из `docs/`. Не изменяй утверждённые правила без явного решения владельца продукта.

## Рабочий процесс

1. Исследуй текущий код до изменений.
2. Для нетривиальной задачи составь план.
3. Делай минимальный законченный вертикальный срез.
4. После изменений запускай format, lint, typecheck и релевантные тесты.
5. Обновляй документацию при изменении API или поведения.
6. Не объявляй задачу завершённой без проверки результата.

## Архитектура

- Модульный монолит, не микросервисы.
- Разделение по бизнес-доменам.
- Запрещены god-object, god-service и универсальные repositories.
- Контроллеры тонкие; бизнес-правила — в domain/application.
- Нет циклических зависимостей.
- Денежные операции атомарны.
- Баланс изменяется только через неизменяемый ledger.
- Idempotency key обязателен для повторяемых платных mutations.
- Деньги хранятся в целых тиынах, не floating point.
- Сервер пересчитывает суммы и проверяет права.
- Скрытые данные не должны утекать через API, логи, файлы и ошибки.
- Исторические финансовые записи и снимок принятого предложения не изменяются.

## Код

- TypeScript strict.
- Без `any` и необоснованных assertions.
- Явные DTO и валидация.
- UTC для дат.
- Типизированные доменные ошибки.
- Не добавляй зависимости без необходимости.
- Не создавай абстракции заранее.

## Обязательные тесты

- публикация заказа списывает 50 ₸ ровно один раз;
- отклик списывает 50 ₸ ровно один раз;
- ИИ-разбор списывает 1 000 ₸ ровно один раз;
- недостаточный баланс блокирует действие;
- один поставщик не создаёт второй отклик;
- принять можно только одно предложение;
- принятие атомарно закрывает заказ;
- контакты доступны только выбранным сторонам;
- исходный AI-файл не выдаётся поставщику;
- повтор заказа создаёт новый платный заказ.

## Безопасность

- object-level authorization;
- rate limiting;
- проверка MIME, сигнатуры, размера и безопасности файлов;
- private storage и временные ссылки;
- блокировка контактов и URL;
- защита от mass assignment;
- audit log административных действий;
- никаких секретов и PII в логах.

## Не добавлять без отдельного решения

- услуги;
- многопозиционные заказы;
- частичные поставки;
- несколько выбранных поставщиков;
- сопровождение сделки или escrow;
- процент с продажи;
- публичные предложения конкурентов;
- название или БИН до принятия;
- автопубликацию результата ИИ;
- обязательную предварительную ручную модерацию.
