# 03. Пользовательские сценарии

## Регистрация
Пользователь создаёт аккаунт, подтверждает контакты, выбирает роль, указывает данные ИП/ТОО. Закрытые реквизиты не показываются до принятия предложения.

## Пополнение
Сумма от 1 000 ₸. Баланс пополняется только после проверенного webhook платёжного провайдера. Повтор webhook не создаёт вторую проводку.

## Ручной заказ
Черновик → обязательные поля → проверка контактов/URL → подтверждение списания 50 ₸ → атомарная публикация.

## Бесплатный файл
Пользователь прикрепляет исходный файл, сам отвечает за несколько товаров, контакты и источник. Исходный файл виден поставщикам. Публикация — 50 ₸.

## Платный ИИ-разбор
Файл проходит техническую проверку → показ цены 1 000 ₸ → списание и постановка job → извлечение/очистка/выявление позиций → выбор одной позиции → ручная проверка → применение к черновику. Исходный файл поставщикам не показывается. Публикация оплачивается отдельно.

## Отклик
Поставщик заполняет предложение → сервер пересчитывает суммы → подтверждение 50 ₸ → атомарное создание ledger entry и offer.

## Редактирование
Изменение активного offer создаёт новую immutable version без повторной оплаты.

## Принятие
Тендерщик подтверждает необратимость → сервер атомарно выбирает одного поставщика, закрывает заказ и раскрывает контакты только выбранным сторонам.

## Повтор
Закрытый заказ копируется в новый черновик без предложений, контактов и результата выбора. Новая публикация — 50 ₸.

## Жалоба
Поставщик указывает причину. Администратор подтверждает или отклоняет. При подтверждении возможны блокировка, санкции и компенсирующая проводка возврата.
