# 08. Безопасность и злоупотребления

## Четыре уровня
1. Структурированные формы.
2. Автоматические ограничения.
3. Жалобы.
4. Реактивная проверка администратором.

Предварительной ручной модерации нет.

## Контакты
Блокировать телефон, email, URL, мессенджеры, соцсети, БИН, название, реквизиты и QR-коды. Использовать normalization, regex/parser, detection обфускации и извлечение текста. Фильтр не считается абсолютным.

## Реклама/услуги
Только товарные категории, обязательные количество/единица, фильтр CTA, fingerprint дублей, rate limits по user/IP/organization.

## Авторизация
RBAC admin, object-level checks, deny by default, public DTO без скрытых полей. UI-маскирование не является защитой.

## Деньги
Webhook signature, idempotency, immutable ledger, reconciliation, запрет отрицательного баланса, audit adjustment, integer minor units.

## Файлы
Allowlist, сигнатура, antivirus, zip-bomb protection, private storage, безопасные имена, security headers.

## Право
Юридические документы, хранение и обработка персональных данных должны быть проверены юристом по законодательству Республики Казахстан до запуска.
